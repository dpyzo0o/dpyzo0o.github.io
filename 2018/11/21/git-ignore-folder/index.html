  • folderToIgnore/ 会排除掉这个文件夹以及文件夹内的所有内容，并且告诉 git 不再处理这个文件夹相关的任何命令
  • folderToIgnore/* 排除的是这个文件夹内的所有内容，但和 folderToIgnore 这个文件夹没有关系。虽然在行为上和上面的写法实际是一样的，就是整个文件夹都不会被追踪，但这是因为 git 本身不追踪空文件夹导致的。当我们想要告诉 git 去追踪某些特定的文件时，这种写法就起作用了。因为执行 folderToIgnore/* 后，git 还是会继续处理这个文件夹的其他命令（如果有的话）
