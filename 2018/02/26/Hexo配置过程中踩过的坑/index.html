Hexo配置过程中踩过的坑

发表于 2018-02-26 | 更新于: 2018-02-27 | 分类于 博客 | 字数统计: 2.3k

今天又折腾了一天在 Hexo 的主题配置上，期间碰到的一些问题在此记录一下。

VS Code 写 Mardown 出现隐藏字符

在写第一篇博文的时候，实际也是自己第一次用 Markdown 完整的写一篇文章，由于最近一直在使用 VS Code，貌似其对 Markdown 的支持也不错，就直接拿来用了。写的过程中确实也比较顺手，实时预览和同步滚动的功能都很不错，在这里推荐一波。在 Mac 上写完、上传、预览都没有什么问题，不过当我在一台 Windows 电脑上打开博客时，就发现文章里面莫名的出现了许多乱码的符号。然后我就在 Mac 上用 Sublime 打开 Markdown 文件查看了一下，找到了问题。出现乱码的地方有样式为”BS”的字符，只不过在 Mac 版 VS Code 里面并没有显示出来，将这些字符删除后 Windows 上就可以正常的显示了。百度了一下发现也有人碰到了这个问题，这个字符其实是 ASCII 码中退格符的 Unicode 表示法，在 Mac 版 VS Code 里切换中英文输入法的过程中产生。目前来看可能是 VS Code 或者 Mac 系统的一个 bug，解决方案是开启 VS Code 的这个选项：

这样 VS Code 里就会显示出这个隐藏的字符，看到时删除就好了。

Mist 主题的 margin 与其它主题不一致

由于自己有一点轻微的强迫症，开始在几个主题间换来换去不知道该用什么，纠结到最后选了 Mist 主题。但又发现了一个问题，就是 Mist 主题里的 <h> 和 <p> 标签显示的前后文间距和其它三个主题不一样，不仔细看其实真的看不出来。不过既然被我发现了，如果不解决的话就浑身难受。利用 Chrome 的开发者工具可以查看标签元素的 CSS 属性，发现就是这两个标签的 margin 设置和另外三个主题不一样，这一点也不 consistent 啊。解决方案就是修改 Mist 主题样式文件，位于 themes/next/source/css/_schemes/Mist/_base.styl。其中有两行代码，将它们注释掉即可。

字数统计插件显示位置

Hexo 提供了一个显示文章和站点字数的插件，不过默认它是显示在文章创建日期的下方的。我觉得不太美观，如果显示在一行会比较简洁，于是就想着修改一下这个显示的位置。方法呢也很简单，找到相应的 layout 设置文件修改即可。具体文件为 themes/next/layout/_macro/post.swig。找到 <div class="post-symbolscount"><div> 的代码块，我们看到这里是 div 标签，块级元素默认是占据一整行的，所以我们将其修改为 <span>，内敛元素就可以和时间在一行显示了。

去掉 social links 前的彩色圆点

这个主题 social links 前加上了随机颜色的彩色圆点，太花哨了，不符合这个主题简洁的整体风格。直接找到 next/layout/source/css/_common/components/sidebar/sidebar-author-links.style，将 &:before 代码块注释掉。
