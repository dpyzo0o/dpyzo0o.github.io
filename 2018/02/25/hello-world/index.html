开始准备写博客的时候，网上找了很多写博客的平台，比如博客园、CSDN、简书、SegmentFault。每一个都注册账号上去试了试，其实各有各的优缺点。博客园和 CSDN 很适合作为技术类的博客，但是这两个的界面太丑了，而且还有广告，简直不能忍。简书和 SegmentFault 的界面很清爽，而且对 Markdown 的支持也很好。本来准备就用其中一个来写了，后来无意间发现了 Hexo + Github Pages 的组合，大致了解了一下，很有 geek 的感觉。又发现很多优秀的前端开发都用了 Github Pages 来搭博客，就随波大流，决定用它了。其实搭建的过程并不复杂，网上也有很多教程供参考，这篇文章就来简单的记录下这个过程。
